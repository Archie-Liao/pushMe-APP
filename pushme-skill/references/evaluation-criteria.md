# 评估标准

## 四维评估框架

每个完成的任务从以下四个维度评估：

### 1. 完成度
- 公式：实际产出 / 原定目标 × 100%
- ≥90%：完成
- 60-89%：基本完成，有遗留
- <60%：未完成，需要判断是继续还是降级

### 2. 效率偏差
- 公式：（实际耗时 - 预估耗时）/ 预估耗时 × 100%
- 正偏差（超时）：分析瓶颈出在哪个步骤
- 负偏差（提前完成）：是否预估过于保守？还是临时砍掉了某些内容？

### 3. 偏差分析
执行中做了哪些与原计划不同的选择？
- 正向偏差（做对了但没计划到的）：记录为未来参考
- 负向偏差（该做但没做的）：分析根因——能力不足？时间不够？判断失误？

### 4. 可复用经验
- 这个任务里学到了什么？
- 下次做同类任务，第一步应该是什么？哪些坑可以跳过？
- 有没有形成可复用的模板/脚本/流程？

## 评估输出格式

```
## 任务评估：[任务名称]
- 完成度：__%
- 效率偏差：实际 __h / 预估 __h（__%）
- 正向偏差：___
- 负向偏差：___
- 可复用经验：___
- 归档：pushme/archive/YYYY-MM-DD-name.md
```

## 周期性效率报告

每周复盘时聚合本周所有完成任务的评估数据：
- 本周完成度均值
- 本周效率偏差均值
- 最常出现的偏差类型
- 本周 vs 上周趋势
