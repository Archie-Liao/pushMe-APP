# Check-in 模板

## 首次访谈

首次激活时，按以下顺序收集信息。每轮最多 2 个问题，不要一次轰炸。

### 第 1 轮：工作基盘
1. 你现在的主业工作是什么？每天固定占用哪些时间段？
2. 主业之外，你在做的项目或学习方向有哪些？（列出名字即可，不用展开）

### 第 2 轮：时间与精力
1. 你一天中精力最好的时间段是？（比如「早上 8-10 点」「晚上 9-11 点」）
2. 有哪些时间段是确定不能被占用的？（通勤、会议、家庭事务等）

### 第 3 轮：优先级
1. 如果接下来一个月你只能完成一件事，你选哪件？
2. 有没有什么任务是你一直在推迟但知道必须做的？

### 第 4 轮：拖延模式
1. 你发现自己最容易在什么情况下拖延？（任务太大？不确定怎么做？害怕做不好？）
2. 你拖延的时候通常在做什么？（刷手机？看视频？做其他简单任务？）

### 第 5 轮：确认与初始化
1. 复述你刚才告诉我的信息，确认无误
2. 创建 `pushme/` 完整文件体系
3. 设定首次 check-in 时间

## 每日 check-in

### 早间 check-in（用户说「早上好」「今天做什么」时）

```
1. 读取当前任务清单
2. 检查：
   - 有没有昨天应该完成但没完成的任务？
   - 有没有 deadline 在未来 2 天内到期的任务？
   - S 级任务最近一次有进度是什么时候？
3. 输出：
   - 今天最重要的 1 件事（必须完成）
   - 如果精力有余，可以做的 1-2 件事
   - 今天绝对不能碰的 1 件事（防止分心）
   - 最小可执行下一步
```

### 晚间 check-in（用户说「复盘」「今天结束了」时）

```
1. 对比今日计划 vs 实际完成
2. 如果有未完成项：
   - 原因是什么？（外部阻塞 / 预估不准 / 逃避）
   - 明天怎么处理？（继续 / 降级 / 删除）
3. 记录到 `check-ins/YYYY-MM-DD.md`
4. 如果今天的实际时间分配和计划偏差超过 50%，追问原因
```

## 每周复盘

```
1. 本周完成了几件事？其中 S 级任务的完成率是多少？
2. 最大的时间浪费在哪里？
3. 哪件事如果本周没做，下周会更难？
4. 下周需要调整什么？（任务优先级 / 时间分配 / 工作方式）
5. 更新 overview.md，清空已完成项，重新排下周优先级
```

## 评估模板

任务完成后使用：

```
## 任务：[名称]
- 完成度：X%（实际产出 / 原定目标）
- 耗时：实际 X 小时 / 预估 Y 小时（偏差 Z%）
- 偏差分析：做了什么与原计划不同的选择？
- 可复用经验：下次怎么做更快？
- 归档到 `archive/YYYY-MM-DD-name.md`
```
