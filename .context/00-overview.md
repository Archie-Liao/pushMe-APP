# pushME 项目概览

## 要解决什么问题

用户（Archie）是 AI 应用层初学者，会用 Claude Code 等 agent，但对编程不熟悉。处于工作学习瓶颈期：本职 + 自学 AI + 日常事务并行，**规划能力强、执行能力弱**，经常失焦，对进度缺乏清晰认识。

需要一个「AI 项目经理」：主动收集信息、拆解可执行方案、监督执行、质疑不合理安排、完成后评估并沉淀记录。

## 产品形态演进

| 阶段 | 形态 | 状态 |
|------|------|------|
| MVP | `pushme-skill/` Codex/Claude Skill + 本地 md 文件体系 | 已有，待真实激活测试 |
| 目标 | 跨端 PWA APP（电脑+手机），云同步 + DeepSeek + 主动推送 | 方案已确认，未开工 |
| 远期 | Capacitor 套壳上架；蒸馏「你自己」接入网站与 APP | 规划中 |

## 人格定位

- 名称：推手 pushME
- 立场：盟友，不是应声虫；会质疑拖延、优先级混乱、反复改计划
- 底层参考：马斯克式质疑 + PM 结构化（见 `pushme-skill/references/pm-framework.md`）

## 用户现有数据

- `任务榜.xlsx`：S/A/B/C 任务（已确认导入 APP）
- `Archie修仙录.xlsx`：早期个人管理 OS，暂不导入
- 修仙主题：APP 支持修仙/现代双主题可切换

## 相关但独立的项目

- **个人网站**（S 级任务）：与知识库/wiki 联系更强，弱耦合 pushME
- **nuwa 蒸馏「你自己」**：原料来自 `journal/`，产物用于网站与 APP 助手

## 核心矛盾（pushME 要解决的）

规划不值钱，执行才值钱。产品价值在「推」和「评估沉淀」，不是再多做一版计划。
